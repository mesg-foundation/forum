Having util packages for testing

Development Engine
ilgooz September 28, 2018, 2:41pm #1

Unit tests are also such programs that require some util packages in the testing process. If we don’t have util packages for testing, the common functionalities we use for testing can be duplicated while testing different packages.

So, I propose having a place in the code base to put some util packages for tests. I think we can have a package in the root of core named as testutils to collect util packages for tests inside.

As a first test util package, I need to extract this one (outputStream) to its own package.
